Project Overview

Welcome to the documentation for Y2Horror - my capstone project that revisits the eerie, glitchy digital aesthetic of early 2000s web horror. This documentation site provides a comprehensive breakdown of the project's conception, design philosophy, technical implementation, and creative decisions.

Quick Stats

  • Project Type: Interactive Web Experience
  • Theme: 2000s Web Horror
  • Development Time: 14 Weeks
  • Technologies: HTML5, CSS3, JavaScript, PHP
  • Special Features: Interactive storytelling, "Cursed" browser effects, Hidden content

The Y2Horror project captures the essence of early internet horror storytelling - a time when the web was still mysterious and untamed. The project deliberately employs the design approaches and technical limitations of the era to create an authentic and immersive experience.

Inspiration & Research

This project was inspired by the unique horror storytelling approach that emerged in the early 2000s internet culture. During this formative period of internet history, creepypastas, forum-based horror stories, and "cursed" websites created a new form of digital folklore.

Key Inspirations

  • Early creepypastas and forum horror stories
  • Alternate reality games (ARGs) of the early 2000s
  • The aesthetic of GeoCities and Angelfire websites
  • "Cursed" image and video content that circulated in the early web
  • Glitch art and datamoshing techniques
  • Low-fi web graphics and the deliberate use of compression artifacts

The project involved extensive research into:

  • Web design trends from 1999-2005
  • Early Flash animation horror content
  • Horror storytelling techniques unique to digital media
  • Technical limitations of early 2000s browsers
  • Visual language of early internet horror

Design & Aesthetics

The visual design of Y2Horror deliberately eschews modern web design principles in favor of an authentic early-2000s aesthetic. The project employs careful use of dated design elements without becoming a parody.

Visual Elements

  • Color Scheme: High-contrast colors, monochromatic text on black backgrounds, red accents for warnings and scares
  • Typography: System fonts like Times New Roman and Courier, deliberately pixelated text
  • Layout: Fixed-width layouts, table-based design, frameset structure in certain sections
  • Graphics: Low-resolution JPEGs, animated GIFs, deliberate compression artifacts
  • UI Elements: Windows 98/XP inspired buttons and dialogs

User Experience Design

The user experience was designed to be deliberately disorienting at times, with navigation that sometimes feels broken or leads to unexpected places. This approach enhances the horror elements by creating a sense of digital unease.

Key Features

Interactive Storytelling

The project tells a fragmented horror narrative through scattered content across various "pages" of the website. Users must explore to piece together the story.

"Cursed" Browser Effects

Special JavaScript effects simulate browser glitches, weird scrolling behavior, and page elements that seem to "watch" or follow the user.

Hidden Content

The site contains numerous secrets only discoverable through specific user actions, viewing page source code, or solving embedded puzzles.

Reactive Environment

Elements of the site react to user behavior, creating the impression that the website itself is sentient or haunted.

Era-Authentic Multimedia

Audio and video content formatted to mimic the quality and style of early 2000s digital media.

  • Hidden Visitor Counter: Tracks actual visitors but occasionally jumps to ominous numbers
  • Cursor Anomalies: Cursor occasionally duplicates or lags when hovering specific elements
  • Message Embedding: Hidden text in page metadata and HTML comments
  • Webcam Spoof: Page that falsely claims to access the user's webcam
  • Audio Trigger: Hidden audio that plays at specific navigation points

Technical Implementation

While the website appears outdated by design, it's built using a combination of modern web technologies that simulate the look and behavior of early 2000s web pages.

Front-End Technologies

  • HTML5: Core structure with deliberate use of deprecated-looking markup
  • CSS3: Used to recreate early 2000s styling without actually using outdated methods
  • JavaScript: Powers the interactive elements and "glitch" effects

Back-End Technologies

  • PHP: Handles user interaction tracking and dynamic content generation
  • Local Storage: Tracks user progress through the experience

Code Example: Glitch Effect

function createGlitchEffect(element) {
    // Original text content
    const originalText = element.textContent;
    
    // Characters for glitch effect
    const glitchChars = '!@#$%^&*()_+-=[];,./\\|{}:"<>?';
    
    // Apply glitch effect
    function applyGlitch() {
        // Only apply glitch sometimes
        if (Math.random() > 0.7) {
            // Create glitched text
            let glitchedText = '';
            for (let i = 0; i < originalText.length; i++) {
                if (Math.random() > 0.9) {
                    glitchedText += glitchChars.charAt(
                        Math.floor(Math.random() * glitchChars.length)
                    );
                } else {
                    glitchedText += originalText[i];
                }
            }
            element.textContent = glitchedText;
            
            // Reset after short delay
            setTimeout(() => {
                element.textContent = originalText;
            }, 100);
        }
    }
    
    // Apply glitch randomly
    setInterval(applyGlitch, 2000);
}

Development Challenges

Creating an authentic 2000s web horror experience while using modern development practices presented several unique challenges:

Challenge Solution
Creating authentic "dated" aesthetics without poor usability Carefully balanced nostalgic design elements with enough modern usability to prevent frustration
Simulating browser quirks without actual browser bugs Created JavaScript routines that mimic the behavior of old browser rendering issues
Making intentional "broken" features while maintaining site stability Implemented controlled chaos - elements that appear broken but function as designed
Creating "unsettling" content while maintaining accessibility Added content warnings and escape options while preserving the horror elements
Balancing nostalgic references with original content Created original story elements that pay homage to early web horror without directly copying

Accessibility Considerations

Despite the intentionally "broken" aesthetic, efforts were made to ensure the site remains accessible:

  • All disturbing content is preceded by warnings
  • Alternative navigation options for users who can't engage with the primary interface
  • Escape buttons on pages with intense effects
  • All critical content remains accessible even when visual effects are active
